Dashboard Table Layout Approach

How should tables be contained within the dashboard panels?

A

Fixed-Height Scrollable Boxes

Every table lives inside a fixed-height container (e.g. 280px). Content overflows with a visible scrollbar inside the box. The dashboard layout stays compact — panels don't grow with data.

Best for: NOC/ops screens where screen space is premium
B

Full Table + Collapsible Sections

Tables render at full natural height (no max-height cap). Entire sections can be collapsed/expanded. Users see all rows without scrolling inside a box — they scroll the page.

Best for: when users need to see full dataset without hunting for a scrollbar
C

Adaptive: Fixed Default, Expand on Demand

Tables start at a fixed height (compact, ~280px scroll). A "Show all" toggle expands the table to full height. Best of both: compact default + full access when needed.

Best for: dashboards used both as quick-glance and deep-dive

# Device Monitoring System - API Routes List
# Updated on 2026-02-07
# NOTE: All APIs below require Authentication.
# TO TEST IN BROWSER: Log in to the application first in the same browser session.
# TO TEST IN POSTMAN: Log in via /login POST, then use the session cookie.

# ==========================================
#  BROWSER TESTABLE JSON APIs (GET)
#  (These return JSON data, easy to view in browser)
# ==========================================

# --- Dashboard Data ---
[GET] /api/dashboard/summary            - Main stats (Device counts, Network health, Active alerts)
[GET] /api/dashboard/top-problems       - Top 5 high latency, packet loss, and recent alerts
[GET] /api/dashboard/alerts             - List of all alerts (Status: active/resolved/all)
[GET] /api/dashboard/trends?range=24h   - Availability & Latency timeline (range: 1h, 24h, 7d, 30d)
[GET] /api/dashboard/inventory          - Vendor & Device Type distribution stats
[GET] /api/dashboard/realtime/interfaces - Top 5 busy interfaces
[GET] /api/dashboard/realtime/network-io - Total Network I/O trend (1h)

# --- Live Tracking Data ---
[GET] /api/tracking/live-summary        - List of tracked devices with online/offline status
[GET] /api/tracking/live-status/<mac>   - Real-time details for a specific device (CPU/RAM/Activity)
                                          (Example: /api/tracking/live-status/00:1A:2B:3C:4D:5E)
[GET] /api/tracking/live-alerts         - Live alerts for tracked devices (high cpu, inactivity)

# --- Monitoring Data ---
[GET] /api/monitoring/status?device_type=all  - Live status of all devices (performs real-time check)
[GET] /api/monitoring/statistics        - Aggregate monitoring stats (Total, Online, Offline)
[GET] /api/monitoring/events            - Recent system events/logs
[GET] /api/monitoring/metrics?device_ip=<ip>&metric_name=network_latency_ms
                                          (Get historical metrics for a device)

# --- Discovery & Scanning ---
[GET] /api/get_local_ip_range           - Returns local network CIDR (e.g., {"ip_range": "192.168.1.0/24"})
[GET] /api/active_scan                  - Returns status of currently running network scan
[GET] /api/scan_progress/<scan_id>      - Progress of specific scan ID

# Device Management (devices.py)
[GET] /devices - Device management page
[POST] /devices/save - Save/Update device (form submit)
[GET] /api/devices - List all devices (JSON)
[GET] /api/devices/<int:device_id> - Get single device details
[POST] /api/devices/<int:device_id> - Update single device details
[GET] /api/devices/<int:device_id>/connections - Agent connection snapshot (top 20) + normalized table rows
                                          Response keys:
                                          - connections (top-20 rows by remote IP, includes remote_hostname + connection_count)
                                          - agent_snapshot.top_remote_ips / unique_remote_ips_count / timestamp
                                          - meta.live_method=agent_snapshot
                                          - meta.snapshot_available / snapshot_age_seconds
                                          - meta.total_connections / total_unique_remote_ips / top_limit
                                          Error shape:
                                          - error.code + error.message (JSON; no HTML errors)
[POST] /api/devices/<int:device_id>/toggle_monitoring - Enable/Disable monitoring
[POST] /api/devices/bulk_add - Bulk add devices
[POST] /api/devices/bulk_delete - Bulk delete devices
[POST] /api/devices/<int:device_id>/update_type - Update device type
[GET] /api/devices/reclassify_all - Re-run classification on all devices

# Device Tracking (tracking.py)
[GET] /tracking - Tracking dashboard (stored employee devices first; add/scan/sync actions)
[GET] /tracking/live - Live tracking wall
[GET] /tracking/history/<int:device_id> - Tracking history page
[POST] /api/tracking/scan - Scan network for running `service.py` agents; auto-save MAC/IP/hostname
[POST] /api/tracking/sync-ips - Rescan + sync IP/hostname; auto-save newly discovered agents
[POST] /api/tracking/save-device - Save manual or discovered tracked device (IP-only save resolves MAC via agent identity)
[POST] /api/tracking/delete-device - Delete tracked device by MAC
[GET] /api/tracking/real-time/<mac_address> - Real-time tracking data
[GET] /api/tracking/live-summary - Live summary of tracked devices
[GET] /api/tracking/live-status/<mac_address> - Live status/metrics for device
[GET] /api/tracking/live-alerts - Live alerts
[Tracking API error contract]
                                          - Handled failures return JSON only (no HTML error pages)
                                          - Standard fields: success=false, error_code, error
                                          - Common codes: DEVICE_NOT_FOUND, AGENT_UNREACHABLE, SESSION_EXPIRED, SERVER_ERROR
[GET] /api/tracking/history/activity/<int:device_id> - Activity history
[GET] /api/tracking/history/resources/<int:device_id> - Resource history
[GET] /api/tracking/history/applications/<int:device_id> - Application history
[GET] /api/tracking/stream/screenshot/<mac_address> - Screen stream
[GET] /api/tracking/stream/camera/<mac_address> - Camera stream

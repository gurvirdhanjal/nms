## 2024-05-24 - [CRITICAL] Hardcoded API Keys and Production Redis Credentials
**Vulnerability:** The application contained a hardcoded tracking API key (`8f42v73054r1749f8g58848be5e6502c`) scattered across multiple files (`config.py`, `service.py`, `server_agent.py`, `routes/file_transfer.py`) and a production Redis connection string with a plaintext password in `config.py`.
**Learning:** Hardcoding secrets directly into source code repositories is a critical security violation. The application had mechanisms to load these from environment variables, but included the production values as fallbacks or defaults, effectively exposing them to anyone with repository access. In `service.py`, `API_KEY` was generated using `generate_secure_keys` but then bypassed by an explicit check for the hardcoded string `8f42v73054r1749f8g58848be5e6502c`.
**Prevention:** Always use secure defaults (like empty strings or localhost equivalents) when fallback mechanisms fail to find environment variables. Never commit production secrets, keys, or credentials to version control. Validate that generated keys are properly utilized and not bypassed by hardcoded development values.
